Contributing a Story

Choose the path that suits you: either contribute directly with Git (recommended), or send your files via DreamForgeGames Discord and a maintainer will help. Both ways are welcome.

Path A — I can use Git (recommended)

If you're comfortable with Git/GitHub, this is the fastest way. You'll add files and a JSON entry yourself.

Go to Git instructions

Path B — I don't want to use Git (Discord)

No problem — send your ZIP and cover in DreamForgeGames Discord and a maintainer will add everything and make the PR for you.

Go to Discord instructions

Path A — Detailed Git workflow

Short checklist: fork/clone → branch → (optionally) add files under /files/your-story/ → edit stories.json → commit & PR.

  1. Install Git (if necessary): Use GitHub Desktop or install Git from git-scm.com. GUI tools are fine.
  2. Fork & clone:
    git clone <repo-url>
  3. Create a new branch:
    git checkout -b add/my-story-name
  4. Add files (optional):

    You may create files/my-story/ and add cover.jpg and my-story.zip. This is optional — you can also provide an external URL in downloadFile.

  5. Edit stories.json: Insert a new object into the stories array. Keep JSON valid (no trailing commas). Example entry: 
    {
  "name": "My Story",
  "description": "Short blurb about the story",
  "imageUrl": "files/my-story/cover.jpg",
  "downloadFile": "files/my-story/my-story.zip",
  "fileSize": "10.5 MB",
  "author": "YourName",
  "version": "1.0",
  "tags": ["adventure"],
  "releaseDate": "2025-10-05"
}
  6. Commit & push:
    git add files/my-story/* stories.json git commit -m "Add: My Story — short description" git push origin add/my-story-name
  7. Create a Pull Request: On GitHub, open a PR from your branch to the repo's main branch. In the PR message include a short description and whether files were uploaded to the repo or sent via Discord.

Quick examples & tips

  • File naming: use lowercase and hyphens, e.g. files/my-story/my-story_v1.0.zip.
  • If the ZIP is >100 MB, use Git LFS or host externally and put the URL in downloadFile.
  • Include author and version fields if possible.
Validation checklist
  • stories.json is valid JSON.
  • downloadFile path exists or points to a valid URL.
  • If you added a cover, imageUrl points to it (or use an external image URL).

Path B — Send files via DreamForgeGames Discord (no Git)

If Git is too much or you'd rather have someone handle the upload, this path is for you. A maintainer will add the files and create the PR.

What to send

  • The story ZIP file (recommended: include version in filename).
  • A cover image (optional but nice): cover.jpg or cover.png.
  • A short JSON snippet or the story details (name, description, author, tags, optional fileSize and releaseDate).

Suggested Discord message (copy & paste)

Hi! I'd like to contribute a story.

Name: My Story
Author: YourName
Description: Short blurb (1-2 lines)
Tags: adventure, puzzle
Files: attached (my-story_v1.0.zip, cover.jpg)
Preferred repo path: files/my-story/
Notes: (optional install notes)

Once a maintainer receives your files they will add them to the repository and create the Pull Request. In the PR we'll include your name and version as you specified.

When to choose this path

  • You don't want to install or use Git.
  • Your files are large and you'd rather a maintainer handle hosting (e.g. Git LFS or external hosting).
  • You prefer a human to check your JSON and file paths before the PR.

Common questions

  • Do I have to upload files to the repo? No — it's optional. You can attach them via Discord instead.
  • What about large files? Use Git LFS or external hosting; tell the maintainer what you used.
  • Can maintainers help me? Yes — if you send files via Discord we will prepare the PR for you.
